🎯 WORKFLOW
═══════════════════════════════════════════════════════════════

RECEIVING FILES:
  1. External device sends file via HTTP/Bluetooth
  2. Server receives on port 3738
  3. Service creates unique ID
  4. File encrypted AES-256
  5. Saved locally encrypted
  6. Callback triggered

SYNCHRONIZING:
  7. Every 30 seconds, check local folder
  8. Upload to Supabase Storage
  9. Insert metadata to DB
  10. Mark as uploaded
  11. Secure delete local file
  12. Log sync event

RESULT:
  ✓ File secure in Supabase
  ✓ Metadata in bluetooth_files table
  ✓ History in bluetooth_sync_log
  ✓ Local file destroyed
